## **WakeWord 测试操作说明文档** 

面向普通用户的设备连接、唤醒测试、录音保存与 ADB 操作指南 

本文档用于指导用户完成 WakeWord 测试应用的基础操作，包括进入系统桌面、打开应用、启动 检测、验证唤醒、保存录音、导出录音文件以及更新 APK。 

## 一、操作前准 **备** 

在进行 WakeWord 测试和录音导出前，请先确认设备线材已正确连接。

**操作说明：** 

1. 连接设备电源线，确保设备正常供电。 

WakeWord 测试操作说明文档 

2. 连接 USB Type-C 数据线，用于设备与电脑之间的数据通信。 

3. 如果后续需要通过 ADB 导出录音文件或更新 APK，请确认 USB Type-C 数据线已连接到电脑， 

并且电脑可以正常识别设备。 

## **注意事项：** 

- 请确认电源线连接牢固，避免测试过程中设备断电。 

- 请使用支持数据传输的 USB Type-C 线，不要使用仅支持充电的线。 

- 通过 ADB 操作前，需要确保电脑已安装 ADB 工具。

## 九、通 **过 ADB 导** 出 **录** 音文件到 **电脑** 

## 录音保存完成后，可以通过 **ADB** 命令将设备中的录音文件导出到电脑。 

图 9：通过 ADB 导出录音文件到电脑 

## **操作步骤：** 

29. 将设备通过 USB Type-C 数据线连接到电脑。 

30. 在电脑上打开命令提示符或终端工具。 

31. 根据 WakeWord 测试界面日志中显示的录音文件路径，执行 ADB 导出命令。 

## **示例命令：** 

adb pull /storage/emulated/0/Android/data/com.wakeword.testapp/files/recordings/20250905_161038_565.pcm D:/pcm 

WakeWord 测试操作说明文档 

## **命令说明：** 

- adb pull：表示从设备中导出文件到电脑。 

- /storage/emulated/0/Android/data/com.wakeword.testapp/files/recordings/20250905_161038_56 

   - 5.pcm：表示设备中的录音文件路径。 

- D:/pcm：表示导出到电脑的目标目录。 

## **结果说明：** 

## 当命令执行成功后，命令行会显示类似以下信息： 

1 file pulled, 0 skipped. 

## 这表示录音文件已成功从设备导出到电脑指定目录。 

## **注意事项：** 

- 每次录音生成的文件名可能不同，请以测试界面日志中显示的实际路径为准。 

- 导出前请确认电脑已安装 ADB 工具，并且设备已正常连接。 

- 如果目标目录不存在，建议先在电脑中创建对应文件夹，例如 D:/pcm。

## 十、通 **过 ADB** 更新 **APK** 

如需在设备上更新 WakeWord 测试应用，可以通过 **ADB** 命令将新的 APK 安装到设备中。如果设备 

中已安装旧版本应用，可使用覆盖安装的方式进行更新。 

图 10：通过 ADB 更新 APK 

## **操作步骤：** 

## 32. 将设备通过 USB Type-C 数据线连接到电脑。 

33. 在电脑上打开命令提示符或终端工具。 

34. 将需要安装的 APK 文件放到电脑本地目录中。 

35. 执行 ADB 安装命令，将 APK 更新到设备。 

## **示例命令：** 

adb install -r D:\WakeWord_Test_1.0.0.apk 

## **命令说明：** 

- adb install：表示安装 APK 到设备。 

- -r：表示覆盖安装，会保留原有应用数据。 

- D:\WakeWord_Test_1.0.0.apk：表示电脑上的 APK 文件路径，请根据实际文件位置修改。 

WakeWord 测试操作说明文档 

## **结果说明：** 

当命令执行成功后，命令行会显示： 

Success 

## 这表示 APK 已成功更新到设备中。 

## **注意事项：** 

- 安装前请确认设备已正确连接到电脑，并且 ADB 可以正常识别设备。 

- 如果电脑连接了多台设备，可以先执行以下命令查看设备列表： 

adb devices 

- 如果更新失败，可检查 APK 路径是否正确，以及设备是否允许通过 ADB 安装应用。
